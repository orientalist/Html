git init

將所在的資料夾交由git控管，會產生.git資料夾。
取消git控管:
直接刪除.git資料夾。

git add [name of file]

將檔案加至暫存區(Staging Area)
  • git add *.Filename Extension:
    將特定副檔名的全部檔案加至暫存區。(Filename Extension=副檔名)
  • git add --all:
    將該資料夾下所有檔案加至暫存區。
  • git add . :
    類似於git add -all，但git add .僅會將目前資料夾下的檔案加入暫存區，而git add --all則是加入整個專案。
*對git而言，任何的修改後的檔案都不會自動加入暫存區，若add檔案後又修改該檔案，則暫存區內的是未修改的檔案。

git commit -m "[Message about commit]"

將在暫存區中所有檔案傳輸至儲存庫(Repository)，即建立了"版本"。

參數

  • git commit -a -m "[Commit Message]":
    等於git add --all 加上 git commit -m ""，但僅對已存在Repository的檔案有效 ，也就是說untracked檔案是無效的。
  • git commit --amend -m "[commit message]":
    修改最近一次commit的文字訊息。
    其他作用:在已commit後將檔案加入該次commit。
    舉例說明:
    git commit -m "Create files"→git add newfile→gti commit --amend --no-edit
    將newfile加入commit訊息為Create files的commit中
