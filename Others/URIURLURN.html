URI URL URN

簡介

URL與URN都是URI的子集。
URI可被視為URL與URN或兩者兼備。URN(統一資源名)如同一個人的名稱，URL則代表一個人的住址。
URN定義某一事物的身分，URL提供尋找該事物的路徑。

URI(Uniform Resource Identifer)統一資源標誌符:

URI是用於標示網路資源的字串，
允許使用者透過特定的協定進行操作互動，常見的形式是URL與URN。
文法:
由 [URI協定名(http、ftp、mailto、file...)][:][協定對應內容] 所構成

URL(Uniform Resource Location)統一資源定位器:

URL是一種URI，標示一個網際網路資源，並指定取得方式。
標準格式:
協定類型:[//伺服器位址[:埠號]][/資源層級UNIX路徑]檔案名[?查詢][#片段ID]
標準格式:
協定類型:[//[存取資源需要的憑證資訊@]伺服器位置[:埠號]][/資源層級UNIX檔案路徑]檔案名[?查詢][#片段ID]
存取資源憑證資訊@、埠號、查詢、片段ID皆屬選填

Http

http使用URL，包含:
  1. 傳送協定(http)
  2. 層級URL標記符號(固定//)
  3. 存取憑證(可省略)
  4. 伺服器(通常為域名/IP)
  5. 埠號(可省)
  6. 路徑(以/分隔)
  7. 查詢(GET模式的表單參數，以?開始，後為Key=value，多組以&連接)，通常以UTF-8編碼
  8. 片段(以#起始)
實例:
http://zh.wilipedia.org:80/w/index.php?title=Special
#備註:URL使用ASCII編碼

URN(Uniform Resource Name)統一資源名稱:

URN是基於某命名空間通過名稱指定資源的URI，無須路徑便可獲取該資源。
